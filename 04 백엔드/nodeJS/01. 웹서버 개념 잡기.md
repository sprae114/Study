## 서버 개발의 기본 원리
모든 **웹 서비스는 요청과 응답**으로 이루어져 있습니다. 

예를 들어
- 웹툰을 보고 싶으면, 네이버 웹툰에 접속하여 웹툰을 요청합니다. 그러면 서버가 웹툰을 보내줍니다.
![[Pasted image 20241009094547.png]]

- 유튜브 영상을 보고 싶으면, 유튜브에 접속하여 영상을 요청합니다. 그러면 유튜브 서버가 영상을 보내줍니다.
![[Pasted image 20241009094553.png]]

이러한 방식이 웹 서비스의 기본 동작 원리입니다.


## 실제 웹사이트 구현 방식✔
실제 사이트들이 어떻게 여러 페이지를 만들어놓았는지 살펴보겠습니다. 예를 들어, 네이버 웹툰 사이트에 접속하면
- `/webtoon`을 붙이면 오늘의 웹툰 페이지를 보여주고,
- `/challenge`를 붙이면 도전 만화 페이지를 보여줍니다.

![[Pasted image 20241011152003.png]]
- 요청 보내기 : `http://localhost:8080/webtoon`

- 요청 받기
```javascript
app.get('${url}', (요청, 응답) => {
	응답.웹페이지_보내기
});
```

클라이언트의 요청을 처리하기 위해 **특정 URL 경로에 대해 어떤 작업을 수행할지를 정의하는 과정**입니다. 이것을 **라우팅**이라고 합니다.


## PORT 개념
컴퓨터는 항상 외부와 통신할 수 있도록 설계되어 있으며, **포트는 외부에서 내 컴퓨터로 접속할 수 있는 구멍**입니다. 기본적으로 사용할 수 있는 포트는 약 6만 개가 있으며, 8080 포트를 열어 외부에서 접속할 수 있게 설정합니다.

## 요청 보내기
- GET 요청: 웹 브라우저의 주소창에 **URL을 입력**하여 쉽게 보낼 수 있습니다.
- POST 요청: HTML **`<form>` 태그를 사용하여 데이터를 서버로 전송**할 수 있습니다.


## RESTful API
RESTful API는 **좋은 API를 설계하기 위한 원칙**입니다.

- 6가지 원칙
1. Uniform Interface: 일관된 URL과 method 사용
2. Client-server 분리: 클라이언트와 서버의 역할 구분
3. Stateless: 각 요청은 독립적으로 처리
4. Cacheable: 서버에서 보낸 데이터는 캐시 가능
5. Layered System: 여러 레이어에서 서버 기능을 구성 가능
6. Code on demand: 서버가 실행 가능한 코드를 전송할 수 있음


## URL 작명 원칙
- **명사 중심**: URL은 동사보다 명사로 구성
- **대시 사용**: 띄어쓰기는 대시(`-`)로 대체
- **확장자 제거**: `.html` 같은 파일 확장자 사용 금지
- **하위 문서**: `/` 기호로 하위 구조 표현

예를 들어,
- `facebook.com/bbc/photos` : BBC 뉴스 페이스북 계정의 사진첩 요청
- `instagram.com/explore/tags/food` : 해시태그 `#food`가 달린 사진 목록 요청
